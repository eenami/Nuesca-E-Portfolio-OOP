Midterm Quizzes

Q: What is Encapsulation in OOP?

A: Encapsulation is the bundling of data (variables) and methods that operate on that data into a single unit or class.

Explanation: It restricts direct access to some of an object's components, which is a means of preventing accidental interference. In Java, this is achieved using private access modifiers and getter/setter methods.

Q: Explain Polymorphism.

A: Polymorphism allows objects to be treated as instances of their parent class rather than their actual class.

Explanation: The most common use of polymorphism in OOP occurs when a parent class reference is used to refer to a child class object. Method overriding is a classic example.

2. Decision Structures & Methods

public static void checkGrade(int score) {
    if (score >= 90) {
        System.out.println("Grade: A");
    } else if (score >= 80) {
        System.out.println("Grade: B");
    } else {
        System.out.println("Grade: C or below");
    }
}

Explanation: A simple method demonstrating the use of `if-else` decision structures to evaluate student scores.

Learnings: Improved my understanding of control flow and how passing arguments to methods enables code reusability.
